# 风格详细规范

本文件包含 7 种风格的完整规范，供 SKILL.md 引用。

---

## ① SCHOLAR — 学术信赖风

**适用项目：** 研究工具、知识管理、隐私优先、自托管

**配色：**
- 主题色：`#2563eb`（权威蓝）
- 数据卡底色：`#f0f4ff`
- 功能卡底色：`#f8fafc`
- 高亮色：`#1d4ed8`
- 强调色：`#dc2626`（仅限关键警告）
- **研究焦点块底色**：`#1e3a5f`（深蓝，用于「研究焦点块」超大数字卡片，仅 SCHOLAR 使用）

**语气特征：**
- 理性、克制、不夸大
- 多用"研究表明""实际测试""数据表明"
- 避免"炸裂""神器""太香了"等夸张词
- 少用感叹号

**钩子模板（3选1）：**
```
A. "当你把文档上传到云端的那一刻——它真的还是你的吗？[项目名] 用 [核心数据] 给出了答案。"
B. "[行业共识] 正在被 [项目名] 改变。它用 [数据] 证明了……"
C. "关于 [领域] 的最佳实践，[X] 万人选择了同一个答案。"
```

**h2 标题模板：**
- "为什么它值得你关注？"（替代"这是什么项目？"）
- "核心技术深度拆解"（替代"它到底有多强？"）
- "谁在用？效果如何？"（替代"效果有多强？"）

**CTA 模板（文末关注引导）：**
```
"📚 如果你也在关注 [领域]，欢迎留言分享你的使用场景和经验～"
"🔍 关注「{{公众号名称}}」，深度拆解更多值得关注的开源研究工具。"
```

**投票 CTA 模板（必须与项目特质绑定，4选1）：**
```
A. 学术研究向："你用哪个方案？[项目名] / [同类A] / [同类B] / 自建方案 → 评论区聊聊你的选型考量"
B. 深度用户向："[项目名] 最打动你的功能是哪个？[功能A] / [功能B] / [功能C] → 评论区分享你的推荐理由"
C. 行业洞察向："你说 [领域] 的下一代方案应该长什么样？[项目名] 的方向对了吗？评论区辩论一下"
D. 知识管理向："你现在的 [知识/研究] 管理流程是什么？[项目名] 解决了你哪个环节的痛点？"
```
→ **每次生成文章时从 4 个模板中随机选 1 个**，禁止连续 3 篇使用同一模板。

**转发金句模板（3选1，30字以内）：**
```
A. "最好的隐私保护，不是相信别人，而是不需要相信别人。"
B. "[数字] 万人的选择，不是因为它免费，而是因为它值得信任。"
C. "当所有数据都属于你自己时，你才真正拥有知识。"
```

**禁用句式：**
- ❌ 禁止用"你有没有想过"开头
- ❌ 禁止用"震惊/炸裂/逆天"等浮夸词
- ❌ 禁止连续 3 段以上纯文字

**代码块配色（强制执行）：**
```
background:#f5f5f5;color:#333;border-left:3px solid #2563eb
```
→ SCHOLAR 面向学术/研究受众，**只允许浅色底代码块**，禁止深色终端风。

**金句质量对照：**
| ✅ 高质量金句 | ❌ 低质量金句 |
|-------------|-------------|
| "最好的隐私保护，不是相信别人，而是不需要相信别人。" | "这是一个很好用的隐私工具。" |
| "当所有数据都属于你自己时，你才真正拥有知识。" | "本项目支持本地存储数据。" |
| "研究不该被垄断——开源让知识回归公共领域。" | "这个项目是开源的，很多人用。" |
> **判断标准**：金句必须是一个**可独立传播的观点**，而非项目功能描述。试着遮住项目名读金句——如果变成一句废话，就不合格。

## ② EXPLORER — 探索发现风

**适用项目：** 数据聚合、趋势洞察、信息挖掘

**配色：**
- 主题色：`#0891b2`（好奇青）
- 数据卡底色：`#ecfeff`
- 功能卡底色：`#f0fdfa`
- 高亮色：`#0e7490`
- 强调色：`#f59e0b`（发现金）
- **探险标签云底色**：`#ecfeff`（与数据卡底色相同）

**语气特征：**
- 好奇心爆棚、像侦探揭密
- 多用"竟然""没想到""原来"
- 强调"搜索""发现""挖掘""背后"等词
- 可用适度感叹号

**钩子模板（3选1）：**
```
A. "如果一个 AI 能同时扫描 12 个平台的真人在说什么，你会问它什么？"
B. "[某热点事件] 发生后，Reddit 上 X 万人讨论了什么？X 平台上的情绪趋势是什么？普通人根本翻不完——但它可以。"
C. "ChatGPT 的回答总是听起来有理但其实缺细节——原因很简单，它不看社交媒体。[项目名] 补上了这个缺口。"
```

**h2 标题模板：**
- "它能搜什么？"（替代"这是什么项目？"）
- "发现了什么？拆开看这 N 个维度"（替代"它到底有多强？"）
- "搜索背后的秘密"（替代"效果有多强？"）

**CTA 模板（文末关注引导）：**
```
"🔍 你最想用它搜哪个平台的什么话题？评论区告诉我！"
"🔍 关注「{{公众号名称}}」，每周发现 GitHub 上最有趣的开源项目。"
```

**投票 CTA 模板（必须与项目特质绑定，4选1）：**
```
A. 场景探索向："你最想用 [项目名] 搜什么？[场景A] / [场景B] / [场景C] / 我有更野的玩法（评论区见）"
B. 对比好奇向："你觉得 [项目名] 比 [同类工具] 好在哪？搜得更多 / 搜得更准 / 界面更好 / 免费无限制"
C. 趋势洞察向："你最近在用哪些工具跟踪行业动态？[项目名] / RSS / Twitter / 其他 → 评论区分享你的信息源"
D. 数据挖掘向："如果 [项目名] 能搜任意数据源，你最想接入哪个？评论区许愿，万一实现了呢"
```
→ **每次生成文章时从 4 个模板中随机选 1 个**，禁止连续 3 篇使用同一模板。

**转发金句模板（3选1，30字以内）：**
```
A. "信息差是最贵的成本，而这个工具在帮你抹平它。"
B. "与其刷 100 条碎片信息，不如让 AI 帮你搜出一个答案。"
C. "真正的搜索，不是找关键词，是发现你不知道自己不知道的东西。"
```

**禁用句式（EXPLORER）：**
- ❌ 禁止用「你有没有」「你敢相信」「你能想象」开头
- ❌ 禁止空洞感叹（"太强了""无敌了"）
- ❌ 禁止只夸不质疑（每个项目至少提 1 个使用限制）

**代码块配色（强制执行）：**
```
background:#f5f5f5;color:#333;border-left:3px solid #0891b2
```
→ ⚠️ **EXPLORER 绝对不能使用深色代码底**（audit 违规案例：grafana 文章错误使用了深色底 `#1e1b4b`）。

**金句质量对照（EXPLORER）：**
| ✅ 高质量金句 | ❌ 低质量金句 |
|-------------|-------------|
| "信息差是最贵的成本，而这个工具在帮你抹平它。" | "这个工具可以搜索多个平台。" |
| "真正的搜索，不是找关键词，是发现你不知道自己不知道的东西。" | "本项目支持跨平台搜索功能。" |
| "与其刷 100 条碎片信息，不如让 AI 帮你搜出一个答案。" | "用这个工具搜索效率很高。" |

## ③ PRAGMATIC — 硬核效率风

**适用项目：** 开发者基础工具、国产开源、性能工具

**配色：**
- 主题色：`#dc2626`（硬核红）
- 数据卡底色：`#fef2f2`
- 功能卡底色：`#fff7ed`
- 高亮色：`#b91c1c`
- 强调色：`#ea580c`（性能橙）

**语气特征：**
- 直接、利落、不废话
- 数字多、对比多、结果导向
- "比 XX 快 3 倍""准确率 96.3%""一行命令搞定"
- 可带国产骄傲感（仅限国产项目）

**钩子模板（3选1）：**
```
A. "你被 PDF 上的文字折磨过多少次？复制出来全是乱码，OCR 工具要么收费要么不准——直到我发现了它。"
B. "[数字] 行代码，[效果]。这不是广告，是 GitHub 上 [X] 万人的选择。"
C. "市面上 [同类工具] 要么 [痛点A]，要么 [痛点B]。[项目名] 是第一个把两者都搞定的。"
```

**h2 标题模板：**
- "一句话：它能做什么？"（替代"这是什么项目？"）
- "硬核拆解：为什么它这么强？"（替代"它到底有多强？"）
- "上手门槛？实测给你看"（替代"效果有多强？"）

**CTA 模板（文末关注引导）：**
```
"⚡ 你还在用什么 OCR/XX 工具？来评论区对比一下！"
"⚡ 关注「{{公众号名称}}」，只推真正好用的开发工具。"
```

**投票 CTA 模板（必须与项目特质绑定，4选1）：**
```
A. 技术选型向："你用哪个方案？[项目名] / [同类A] / [同类B] / 自己写的 → 评论区晒你的技术栈"
B. 性能党向："对你来说，选工具的首要考虑是？速度 / 免费 / 开源 / 生态 → 评论区站队"
C. 效率对比向："用了 [项目名] 之后，你省了多少时间？10分钟 / 1小时 / 半天 / 不敢想没有它的日子"
D. 开发者吐槽向："你被 [同类工具] 坑得最惨的一次是什么？换了 [项目名] 之后好了吗？评论区吐槽大会开始"
```
→ **每次生成文章时从 4 个模板中随机选 1 个**，禁止连续 3 篇使用同一模板。

**转发金句模板（3选1，30字以内）：**
```
A. "[X] 行命令，搞定你过去要配半天的环境。没有废话。"
B. "速度不是它唯一的标签——无依赖、零配置、一行装好。"
C. "开源世界最浪漫的事：一个工具，解决一个真实痛苦。"
```

**禁用句式（PRAGMATIC）：**
- ❌ 禁止用"你有没有被 XX 折磨过"开头
- ❌ 禁止虚假对比（无 benchmark 数据支撑）
- ❌ 禁止吹嘘"完爆/碾压"（可用"比 XX 快 X 倍，实测数据见……"）

**代码块配色（强制执行）：**
```
background:#1e1b4b;color:#e2e8f0
```
→ PRAGMATIC 面向硬核开发者，使用**深色终端风**代码块，体现技术感。

**金句质量对照（PRAGMATIC）：**
| ✅ 高质量金句 | ❌ 低质量金句 |
|-------------|-------------|
| "3 行命令，搞定你过去要配半天的环境。没有废话。" | "这个工具安装很方便。" |
| "开源世界最浪漫的事：一个工具，解决一个真实痛苦。" | "这是个好用的开源工具。" |
| "速度不是它唯一的标签——无依赖、零配置、一行装好。" | "它很快，也很好用。" |

## ④ MENTOR — 导师前辈风

**适用项目：** 学习路线、教程、面试、职业成长

**配色：**
- 主题色：`#7c3aed`（温暖紫）
- 数据卡底色：`#f5f0ff`
- 功能卡底色：`#faf5ff`
- 高亮色：`#6d28d9`
- 强调色：`#f59e0b`（成就金）

**语气特征：**
- 温暖、真诚、像过来人倾囊相授
- 强调故事性、真实经历
- 不卖焦虑，不画大饼
- "如果你也在 [阶段]，这份 [资料] 可能帮到你"

**钩子模板（3选1）：**
```
A. "X 年前，[作者背景]。他花了 X 个月自学 [技能]，拿到了 [成果]。后来他把学习笔记开源了，[X] 万人因此受益。"
B. "如果你正在准备 [XX 面试/XX 考试/XX 自学]，大概率已经被各种攻略淹没了。但有一份资料，是作者用亲身经历验证过的。"
C. "[名人/作者] 曾经也是一个 [普通人]。他做过的事，你也可以。"
```

**h2 标题模板：**
- "它的来历？一个真实的故事"（替代"这是什么项目？"）
- "里面有什么？帮你省半年的精华"（替代"它到底有多强？"）
- "适合谁？不适合谁？"（替代"这个工具适合你吗？"）

**CTA 模板（文末关注引导）：**
```
"💬 你现在在哪个阶段？自学的困惑是什么？来评论区聊聊，一起加油！"
"🧡 关注「{{公众号名称}}」，不止推工具，更陪你成长。"
```

**投票 CTA 模板（必须与项目特质绑定，4选1）：**
```
A. 成长阶段向："你现在处于哪个阶段？刚开始自学 / 准备面试 / 已经上岸 / 回头看资料 → 评论区找到你的同路人"
B. 学习方式向："你学新技术时更依赖什么？看书 / 看视频 / 实战项目 / 开源资料（如 [项目名]）→ 分享一下你的学习方法"
C. 经验传承向："如果让你给 1 年前的自己推荐 1 份学习资料，你会推荐 [项目名] 吗？为什么？评论区聊聊"
D. 瓶颈互助向："你自学时最大的卡点是？看不懂源码 / 记不住概念 / 没有项目练手 → 评论区一起解决"
```
→ **每次生成文章时从 4 个模板中随机选 1 个**，禁止连续 3 篇使用同一模板。

**转发金句模板（3选1，30字以内）：**
```
A. "最好的学习资料，不是最贵的，是一个过来人掏心掏肺的笔记。"
B. "[X] 万人已经走过的路，你不必独自摸索。"
C. "自学最怕的不是进度慢，是方向错。这份资料帮你看清路。"
```

**禁用句式（MENTOR）：**
- ❌ 禁止贩卖焦虑（"你再不学就晚了""同龄人已经……"）
- ❌ 禁止夸大效果（"看完就能拿 Offer"）
- ❌ 禁止用"你有没有迷茫过"开头（太套路）

**代码块配色（强制执行）：**
```
background:#f5f5f5;color:#333;border-left:3px solid #7c3aed
```
→ MENTOR 面向学习成长者，使用**浅色低压力**代码块，降低阅读门槛。

**金句质量对照（MENTOR）：**
| ✅ 高质量金句 | ❌ 低质量金句 |
|-------------|-------------|
| "最好的学习资料，不是最贵的，是一个过来人掏心掏肺的笔记。" | "这份学习资料很好。" |
| "自学最怕的不是进度慢，是方向错。这份资料帮你看清路。" | "这个项目帮你学习编程。" |
| "几万人已经走过的路，你不必独自摸索。" | "很多人用这个学习。" |

C. "当技术突破的瞬间，聪明人已经悄悄关注了。"
