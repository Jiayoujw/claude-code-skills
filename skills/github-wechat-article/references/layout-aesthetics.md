# 微信公众号排版美学指南

> 本文件为 `SKILL.md`「排版美学原则」章节的完整展开，是公众号文章视觉设计的权威参考。

---

## 一、设计哲学

微信公众号文章不是网页，也不是 PDF。它有三个不可改变的前提：

1. **阅读设备是手机**：375px 宽，竖屏滑动，拇指控制一切
2. **微信编辑器会过滤 CSS**：只有 `<table>`/`<p>`/`<h1>`/`<h2>`/`<blockquote>`/`<pre>`/`<img>` + 内联 style 是安全的
3. **读者注意力极短**：3 秒扫读决定是否继续，30 秒决定是否读完

这三条前提决定了我们所有的设计决策。任何违反这三条前提的"好看"都是自嗨。

## 二、视觉层级系统

### 2.1 五级字号体系

读者扫读一篇文章只需 3-5 秒。号主必须在这几秒内建立清晰的视觉层级：

| 层级 | 元素 | 字号 | 字重 | 颜色 | 在微信中的角色 |
|------|------|------|------|------|--------------|
| L0 | 封面图 | — | — | — | 第一视觉冲击，决定点不点进来 |
| L1 | h1 主标题 | **24px** | 默认 | `#1a1a1a` | "我在读什么？"——进来看的第一件事 |
| L2 | h2 章节标题 | 18-19px | 默认 | `#1a1a1a` | 扫读锚点，翻到就知道这段讲什么 |
| L3 | 关键数据/金句 | 16-48px | 700+ | 主题色 | 眼睛自然停留的数个字/一句话 |
| L4 | 正文段落 | 14-15px | 默认 | `#333` | 详情阅读区 |
| L5 | 辅助信息 | 12-13px | 默认 | `#666` / `#999` | 星数、日期、引导语 |

**核心规则**：
- 相邻层级字号差 ≥ 2px（否则层级模糊）
- h1 固定 24px，不随风格变化（这是 6 批次 audit 持续发现的 P0 问题）
- 正文固定 15px（SCHOLAR 可 16px），不因"想多塞内容"而缩小

### 2.2 字重层级

微信只支持 `normal` 和 `bold`（通过 `<strong>` 或 `font-weight:700`），不支持 300/500/600 等中间字重。

字重使用规则：
- `<strong>` 仅用于：关键词（每段 ≤ 3 个）、功能名、卡片标题
- 禁止整句加粗（会失去强调效果）
- 禁止正文段落全部用 `<strong>`（等于没加粗）

### 2.3 h2 装饰的 7 种风格差异化

每种风格的 h2 装饰方式不同，这是 7 种风格视觉差异的第一体现：

| 风格 | h2 装饰方式 | 示例 CSS |
|------|-----------|---------|
| SCHOLAR | 无装饰（纯文字） | `margin:30px 0 18px; font-size:18px` |
| EXPLORER | 左侧色条 | `border-left:4px solid #0891b2; padding-left:12px` |
| PRAGMATIC | 左侧色条 | `border-left:4px solid #dc2626; padding-left:12px` |
| MENTOR | 左侧细色条 | `border-left:3px solid #7c3aed; padding-left:10px` |
| FUTURIST | 全宽浅色背景条 + 居中 | `background:#e0e7ff; text-align:center; padding:8px 0` |
| ARCHITECT | 左侧色条 | `border-left:4px solid #0f172a; padding-left:14px` |
| COMPANION | 左侧细色条 | `border-left:3px solid #059669; padding-left:10px` |

## 三、色彩系统详解

### 3.1 三色法则

每篇文章只使用 **3 种功能性颜色**，不多不少：

| 颜色角色 | 固定值 | 说明 |
|---------|--------|------|
| 正文色 | `#333` | 永远不用纯黑 `#000`（太刺眼）；不用 `#666` 做正文（太淡） |
| 辅助灰 | `#666` / `#999` | `#666` 用于次级标签；`#999` 用于日期、来源、引导语 |
| 主题色 | 来自风格配色表 | 全文唯一的彩色，用于 h2 装饰、关键数字、CTA、投票 |

**禁止事项**：
- ❌ 同一篇文章出现 2 种主题色（如同时出现红色和蓝色强调）
- ❌ 用主题色写大段正文（大面积彩色 = 视觉疲劳）
- ❌ 用低对比度配色（如浅灰字配白底，手机上根本看不清）

### 3.2 7 种风格配色完整表

| 风格 | 主题色 | 数据卡底色 | 功能卡底色 | 高亮色 | 强调色 | 分割线色 |
|------|--------|-----------|-----------|--------|--------|---------|
| SCHOLAR | `#2563eb` | `#f0f4ff` | `#f8fafc` | `#1d4ed8` | `#dc2626` | `#d1d5db` |
| EXPLORER | `#0891b2` | `#ecfeff` | `#f0fdfa` | `#0e7490` | `#f59e0b` | `#0891b2` (dashed) |
| PRAGMATIC | `#dc2626` | `#fef2f2` | `#fff7ed` | `#b91c1c` | `#ea580c` | `#fecaca` |
| MENTOR | `#7c3aed` | `#f5f0ff` | `#faf5ff` | `#6d28d9` | `#f59e0b` | `#d1d5db` |
| FUTURIST | `#1e1b4b` | `#e0e7ff` | `#eef2ff` | `#4338ca` | `#06b6d4` | `#1e1b4b` (2px solid) |
| ARCHITECT | `#0f172a` | `#f1f5f9` | `#f8fafc` | `#334155` | `#0284c7` | `#cbd5e1` |
| COMPANION | `#059669` | `#ecfdf5` | `#f0fdf4` | `#047857` | `#f97316` | `#d1d5db` |

### 3.3 背景色区分度检查

数据卡底色和功能卡底色必须肉眼可区分。快速检查：两者亮度差 ≥ 5%。

| 风格 | 数据卡底色 | 功能卡底色 | 亮度差 | 是否合格 |
|------|-----------|-----------|--------|---------|
| SCHOLAR | `#f0f4ff` (≈96%) | `#f8fafc` (≈98%) | 2% | ⚠ 临界，在 OLED 屏上可能看不出 |
| EXPLORER | `#ecfeff` (≈99%) | `#f0fdfa` (≈99%) | 0% | ⚠ 几乎看不出，建议 EXPLORER 功能卡改用 `#f5f5f5` |

→ **如果两张卡片在手机上看不出区别，读者会以为排版出错了。**

### 3.4 颜色可访问性

- 主题色与白色背景的对比度 ≥ 4.5:1（WCAG AA 标准）
- `#dc2626`（PRAGMATIC 红）配白底 → 5.3:1 ✅
- `#0891b2`（EXPLORER 青）配白底 → 4.6:1 ✅
- `#059669`（COMPANION 绿）配白底 → 4.5:1 ✅（临界）

## 四、间距节奏系统

### 4.1 4-12-20 节奏

微信文章没有 CSS margin-collapse。所有间距靠手写 margin 逐行控制。

```
┌─────────────────────────────────────┐
│ h2 标题                              │
│ margin-top: 20-32px                  │  ← 大间距：章节间呼吸
│ margin-bottom: 12-18px               │  ← 中小间距：标题→内容归属感
├─────────────────────────────────────┤
│ 段落 p                               │
│ margin: 0 0 12-18px                  │  ← 中间距：段落间节奏
├─────────────────────────────────────┤
│ 表格 table                           │
│ margin: 10-18px 0                    │  ← 中间距：卡片→文字过渡
├─────────────────────────────────────┤
│ 分割线 hr                            │
│ margin: 20-32px 0                    │  ← 大间距：章节分割
└─────────────────────────────────────┘
```

### 4.2 h2 上大下小原则

h2 的 margin 必须 **上大下小**：
- ✅ `margin: 28px 0 14px`（正确：标题带领内容）
- ❌ `margin: 14px 0 28px`（错误：标题和上文粘在一起，和下文割裂）
- ❌ `margin: 20px 0 20px`（临界：对称间距无法体现归属关系）

### 4.3 各风格间距配置

| 风格 | h2 margin-top | h2 margin-bottom | 段间距 | 行高 |
|------|-------------|-----------------|--------|------|
| SCHOLAR | 30px | 18px | 18px | 1.9 |
| EXPLORER | 20px | 12px | 12px | 1.7 |
| PRAGMATIC | 20px | 10px | 10px | 1.6 |
| MENTOR | 24px | 16px | 14px | 1.8 |
| FUTURIST | 28px | 16px | 14px | 1.7 |
| ARCHITECT | 24px | 14px | 14px | 1.8 |
| COMPANION | 22px | 14px | 12-18px | 1.75 |

### 4.4 间距密度与文章长度的关系

| 文章长度 | 推荐间距策略 | 理由 |
|---------|------------|------|
| < 800 字 | 疏朗（SCHOLAR 级） | 短文章需要留白撑场 |
| 800-1500 字 | 适中（MENTOR 级） | 标准节奏 |
| > 1500 字 | 紧凑（PRAGMATIC 级） | 长文章太疏朗会让读者觉得"怎么还没完" |

## 五、留白的力量

### 5.1 四个不可侵犯的留白区

1. **文章开头**：h1 之后、封面图之前 → 至少 8px（`margin:0 0 8px` 到下一个元素）
2. **文章结尾**：最后一段文字 → "点赞引导"之间 → 至少 16px
3. **表格卡片内**：文字不能贴边 → cellpadding/cellspacing ≥ 10px → 或 td padding ≥ 10px
4. **分割线上下**：至少 20px（分割线不是贴上去的，是有呼吸空间的）

### 5.2 留白密度与风格的关系

| 风格 | 留白密度 | 心理暗示 |
|------|---------|---------|
| SCHOLAR | 高留白 | "这篇文章值得认真读" |
| FUTURIST | 戏剧化留白 | "这是大事，你要停下来感受" |
| PRAGMATIC | 低留白 | "快看，全是干货" |
| COMPANION | 跳跃留白 | "像朋友聊天，想到哪说到哪" |

### 5.3 常见留白错误

| 错误 | 后果 | 修复 |
|------|------|------|
| 封面图和下一段文字之间无间距 | 图片和文字糊在一起 | 至少 `margin:16px 0` |
| 表格内文字 padding=4px | 文字贴边，阅读不适 | 改为 ≥ 10px |
| 分割线直接贴在 h2 或 p 上 | 视觉上"挤" | 上下各加至少 20px margin |
| CTA 投票区和底部引导之间无间距 | 读者不知道投票结束了 | CTA table 下加 `margin-bottom:16px` |

## 六、卡片设计原则

### 6.1 卡片三要素

微信中做卡片唯一可靠的方式是 `<table>`。每张卡片应包含三层信息：

```
┌─────────────────────────────────────┐
│ 卡片标签（12-13px, #666）            │  ← 轻量：这是什么卡片？
│                                     │
│ 卡片核心（14-48px, 主题色/#333, 700）│  ← 焦点：最想让读者看到的东西
│                                     │
│ 卡片辅助（12px, #999）               │  ← 补充：不读也不影响理解
└─────────────────────────────────────┘
```

### 6.2 数据卡片 vs 功能卡片

| 属性 | 数据卡片 | 功能卡片 |
|------|---------|---------|
| 背景色 | 数据卡底色 | 功能卡底色 |
| 核心内容 | 数字/指标（Star 数、性能数据） | 文字描述（功能说明、对比） |
| 字号 | 大数字 22-48px | 正文 14-15px |
| 排列 | 网格排列（2-4 列） | 列表/对比排列 |
| 出现频率 | 每篇 1 个 | 每篇 2-4 个 |

**⚠️ 数据卡片和功能卡片的背景色必须不同**（分别用风格指定的底色）。

### 6.3 卡片数量上限

| 文章长度 | 卡片最多数量 | 原因 |
|---------|------------|------|
| < 800 字 | 3 个 | 卡片太多 → 没有主次 |
| 800-1500 字 | 5 个 | 标准上限 |
| > 1500 字 | 6 个 | 绝对上限，再多读者麻木 |

### 6.4 卡片排列节奏

连续 2 张卡片之间必须有 ≥ 1 段文字过渡（至少 1 句），不能卡片连卡片。

```
✅ 正确：卡片 → p 过渡段 → 卡片 → p 过渡段 → 卡片
❌ 错误：卡片 → 卡片 → 卡片（读者：这到底是文章还是 Excel？）
```

## 七、移动端优化

### 7.1 屏幕尺寸适配

微信正文区域在 iPhone 14 Pro 上约 375px 宽。所有设计基于此：

- 表格列数：≤ 4 列（5 列以上在手机上挤成面条）
- 图片：`max-width:100%`（不写宽度，自适应）
- 代码块：字号 ≥ 13px（太小需要双指放大）
- 投票选项：每行一个选项，不要横排（手机上一行放不下 3 个选项）

### 7.2 可点击区域

手机上是手指操作，不是鼠标。可交互元素的最小点击区域 44×44px（Apple HIG 标准）。

微信文章中的"可点击"主要是心理层面的投票选项：
- 投票选项之间至少 6px 间距（防止误触心理）
- 选项文字 ≥ 14px（太小看不到）

### 7.3 一屏内容规划

微信阅读是连续的，没有"翻页"概念。但一屏（约 500px 滚动高度）应包含：

- 至少 1 个视觉锚点（见原则 7）
- 不超过 3 段纯文字
- 如果这段内容读者扫过去可能跳过 → 加一个表格/图片/引用块打断

## 八、视觉动线设计

### 8.1 F 型扫读模式

读者在手机上扫读微信公众号文章时，遵循 F 型模式：
1. 第一行（标题）→ 从左到右完整阅读
2. 往下扫 → 左侧对齐线（h2 装饰、blockquote 左侧色条）成为导航
3. 再往下 → 只扫读左侧几个字，右侧余光掠过

**设计启示**：
- h2 标题的前 4-6 个字必须是信息量最高的词
- 关键信息放在段落左侧（blockquote 天然适合）
- 大数字放表格中间（居中），读者扫到时会自动停下

### 8.2 Z 型浏览模式

当页面有大幅图片时，读者会切换为 Z 型：
1. 图片 → 扫一眼
2. 图片下方第一行文字 → 从左到右读
3. 继续往下 → 扫左侧

**设计启示**：
- 图片下方第一句话极其重要（必须是有信息量的，不能是废话过渡）
- 图片不要放在两个 h2 之间无人认领的位置

### 8.3 视觉节奏曲线

好的文章像一首歌，有主歌有副歌，有快有慢：

```
开头（钩子）   → 快节奏、短段落、制造紧迫感  [相当于前奏]
封面图         → 呼吸点                        [相当于间奏]
项目介绍       → 中等节奏、建立认知            [相当于主歌]
功能拆解       → 密集节奏、信息轰炸            [相当于副歌]
上手/适合谁用  → 放缓、让读者消化              [相当于桥段]
金句 + CTA     → 高潮 + 收尾                   [相当于尾声]
```

## 九、各风格视觉差异化策略

### 9.1 SCHOLAR — 论文式视觉

| 维度 | 策略 |
|------|------|
| 配色心理 | 蓝色 = 信任、理性、学术。大面积留白制造"值得认真对待"的氛围 |
| 字号选择 | 正文 16px（唯一用 16px 正文的风格，强调"值得读"） |
| 卡片策略 | 尽量少用卡片，数据融入段落。如果必须用，只用 1 个数据卡 |
| 装饰克制 | h2 无装饰线，分割线极简（`#d1d5db` 1px），blockquote 字号与正文相同 |
| 典型翻车 | 卡片太多 → 从"论文"变成"宣传册" |

### 9.2 EXPLORER — 地图式视觉

| 维度 | 策略 |
|------|------|
| 配色心理 | 青色 = 好奇、发现。虚线分割线像地图上的探险路线 |
| 卡片策略 | 横向标签云（3-4 列紧凑排列），每列一个搜索源 |
| 功能展示 | blockquote 标记"🔍 搜索维度"，像探险日志的条目 |
| 代码块 | 浅色底（强制），不能让代码块压过探索的轻快感 |
| 典型翻车 | 深色代码底 → 破坏了探索的轻盈感（audit 违规案例：grafana） |

### 9.3 PRAGMATIC — 仪表盘式视觉

| 维度 | 策略 |
|------|------|
| 配色心理 | 红色 = 效率、行动、紧迫感。紧凑间距制造"全是干货"的感觉 |
| 卡片策略 | 2×2 数据网格最前面，前后对比表主力展示 |
| 功能展示 | 「没有它时 → 有了它之后」对比表，制造戏剧化反差 |
| 代码块 | 深色终端风（`#1e1b4b`），开发者看了亲切 |
| 典型翻车 | 间距太疏朗 → 失去了效率感 |

### 9.4 MENTOR — 书信式视觉

| 维度 | 策略 |
|------|------|
| 配色心理 | 紫色 = 温暖、智慧、成长。柔和圆润的视觉语言 |
| 卡片策略 | 尽量不用独立的正式卡片，用 blockquote 代替 |
| h2 装饰 | 左侧细线（3px），比 PRAGMATIC 的 4px 更柔和 |
| 典型翻车 | 数据卡片太大太硬 → 从"前辈的信"变成"产品说明书" |

### 9.5 FUTURIST — 宣言式视觉

| 维度 | 策略 |
|------|------|
| 配色心理 | 深蓝 = 宇宙、未来、深邃。戏剧化间距制造"暂停感受"的节奏 |
| 数据卡 | 巨型单数字（48px Star 数），震撼第一眼 |
| h2 装饰 | 全宽浅色背景条 + 居中文字，像宣言的章节标题 |
| 分割线 | 粗实线 2px，`#1e1b4b`，比 1px 细线更有宣言的力量感 |
| 功能展示 | 「以前 → 现在」认知对比表（不是 PRAGMATIC 的前后对比表！） |
| ⚠ 典型翻车 | 用 PRAGMATIC 的「前后对比表」和「硬核拆解」h2 → audited：pytorch 文章串味 |

### 9.6 ARCHITECT — 蓝图式视觉

| 维度 | 策略 |
|------|------|
| 配色心理 | 深灰蓝 = 权威、标准、底层。冷峻专业的工业感 |
| h2 装饰 | 左侧色条（4px），配合更大的 padding-left（14px），制造"规格说明"的正式感 |
| 代码块 | 最深底色（`#0f172a`），比 PRAGMATIC 更深，体现底层感 |
| 卡片策略 | 结构化数据表，信息密集但有序 |
| 典型翻车 | 用暖色或活泼的语气 → 破坏了"标准制定者"的严肃感 |

### 9.7 COMPANION — 聊天式视觉

| 维度 | 策略 |
|------|------|
| 配色心理 | 绿色 = 亲切、自然、好用。轻快的视觉节奏 |
| 卡片策略 | 「第一印象反应卡」：emoji + 简短标签，不像正式数据 |
| 功能展示 | 段落 + `<strong>` 高亮功能名，不用正式卡片 |
| 间距 | 跳跃式（12-18px 不等），模拟聊天记录的节奏 |
| 典型翻车 | 过于正式的卡片布局 → 失去"朋友安利"的亲切感 |

## 十、常见排版误区

### 10.1 误区：颜色越多越好看
- ❌ 一篇文章出现 3 种以上彩色强调
- ✅ 正文只用一种主题色，灰度只用 `#333` / `#666` / `#999` 三级

### 10.2 误区：卡片越多越专业
- ❌ 连续 3 张卡片无文字过渡
- ✅ 卡片之间有至少 1 句过渡文字

### 10.3 误区：代码块统一深色底
- ❌ 所有风格都用 `background:#1e1b4b`（audit 发现 EXPLORER 违规）
- ✅ 按代码块配色强制映射表选择

### 10.4 误区：字号小了能多塞内容
- ❌ 正文 13px（手机上根本看不清，读者直接划走）
- ✅ 正文 14-15px，SCHOLAR 可 16px

### 10.5 误区：分割线随便放
- ❌ 分割线上下间距不足，贴在上面段落和下面段落上
- ✅ 分割线作为章节分界，上下各至少 20px 留白

### 10.6 误区：标题装饰全一样
- ❌ 所有风格 h2 都用 `border-left:4px solid {{主题色}}`
- ✅ 7 种风格 7 种 h2 装饰方式（见 2.3 节）

### 10.7 误区：金句写成项目描述
- ❌ "这是一个支持多平台的搜索工具"（这是项目描述，不是金句）
- ✅ "信息差是最贵的成本，而这个工具在帮你抹平它。"（这是观点，可以独立传播）

## 十二、视觉自查工具速查

快速判断一篇文章的视觉质量，只需回答 5 个问题：

1. **3 秒测试**：不读文字，快速滑动——能看到几个不同的颜色区域？（答：至少 3 个，封面图/数据卡/CTA 区各不同色）
2. **h2 测试**：只看 h2 标题，能知道这篇文章讲了什么吗？（答：应该能。如果 h2 是"为什么""拆解""谁在用"等模板标题，说明太通用，需要加入项目关键词）
3. **金句测试**：遮住项目名，金句还有意义吗？（答：应该是一句可独立传播的观点）
4. **卡片测试**：每张卡片的背景色都不一样吗？（答：数据卡和功能卡背景色应不同）
5. **代码块测试**：代码块的配色和文章主色调和谐吗？（答：浅色风格用浅色底，深色风格用深色底）

## 十三、炫酷视觉技术（WeChat 极限挑战）

> 微信公众号编辑器严禁 CSS3 特性（无渐变、无阴影、无圆角、无动画、无 flex/grid）。
> 以下所有"炫酷"效果均通过 `<table>` + 内联 style 的创造性组合实现。

### 13.1 大数字冲击技术

**原理**：字号越大，视觉权重越高。在 WeChat 375px 手机上，28-56px 的数字会强制读者停下滑动。

**7 种风格的大数字策略**：

| 风格 | 字号 | 底色 | 应用位置 | HTML 骨架 |
|------|------|------|---------|-----------|
| FUTURIST | 56px | `#1e1b4b` | 封面图后第一个卡片 | `<p style="font-size:56px;font-weight:900;color:#fff;">` |
| SCHOLAR | 42px | `#1e3a5f` | 「研究焦点块」 | `<p style="font-size:42px;font-weight:800;color:#fff;">` |
| PRAGMATIC | 28px | `#fef2f2` | 2×2 数据网格 | `<p style="font-size:28px;font-weight:800;color:#dc2626;">` |
| ARCHITECT | 15px (bold) | `#0f172a` | 规格说明书卡 | 不用超大数字，用深色底白字做 label:value |
| EXPLORER | 26px emoji | `#ecfeff` | 探险标签云 | `<p style="font-size:26px;margin:0;">📱</p>` |
| MENTOR | 15px (bold) | 渐变色表 | 故事时间线卡 | 强调故事性，不强调数字 |
| COMPANION | 28px emoji | `#ecfdf5` | 第一印象反应卡 | `<p style="font-size:28px;margin:0;">😍</p>` |

**实战示例 — FUTURIST 巨型数字**：
```html
<table width="100%" cellpadding="28" cellspacing="0" border="0" style="background:#1e1b4b;margin:24px 0;text-align:center;">
  <tr><td>
    <p style="font-size:56px;font-weight:900;color:#ffffff;margin:0;line-height:1;">85k</p>
    <p style="font-size:15px;color:rgba(255,255,255,0.75);margin:12px 0 0;">GitHub Stars · AI 框架的变革正在发生</p>
  </td></tr>
</table>
```

### 13.2 色彩对比撕裂技术

**原理**：在同一张 table 内使用两种差异极大的背景色，制造视觉上的"撕裂感"，让对比内容产生戏剧化效果。

**PRAGMATIC — 前后对比撕裂卡**：
```html
<table width="100%" cellpadding="10" cellspacing="0" border="0" style="margin:12px 0;">
  <tr><td style="font-size:13px;color:#999;background:#f5f5f5;padding:8px 12px;">❌ 没有它时</td></tr>
  <tr><td style="font-size:14px;color:#555;background:#f5f5f5;padding:8px 12px;">手动复制粘贴，格式全乱……</td></tr>
  <tr><td style="font-size:13px;color:#dc2626;font-weight:700;background:#fef2f2;padding:8px 12px;">✅ 有了 PDF-extract</td></tr>
  <tr><td style="font-size:14px;color:#333;background:#fef2f2;padding:8px 12px;">一键提取，完美保留表格和格式。</td></tr>
</table>
```

**FUTURIST — 认知颠覆对比卡（左灰右蓝）**：
```html
<table width="100%" cellpadding="10" cellspacing="0" border="0" style="margin:14px 0;">
  <tr>
    <td style="font-size:13px;color:#999;background:#f5f5f5;padding:10px 12px;">以前我们以为</td>
    <td style="font-size:13px;color:#e0e7ff;font-weight:700;background:#1e1b4b;padding:10px 12px;">PyTorch 告诉我们</td>
  </tr>
  <tr>
    <td style="font-size:14px;color:#555;background:#f5f5f5;padding:10px 12px;">AI 训练需要超级计算机集群</td>
    <td style="font-size:14px;color:#cbd5e1;background:#1e1b4b;padding:10px 12px;">一台 GPU 笔记本就能跑起来。</td>
  </tr>
</table>
```

### 13.3 深色底卡片技术（仅 FUTURIST / ARCHITECT / PRAGMATIC 代码块）

**原理**：微信正文默认白底。在白底文章中插入一块深色底卡片，制造强烈的明暗对比。

| 风格 | 深底色值 | 文字色 | 使用场景 |
|------|---------|--------|---------|
| FUTURIST | `#1e1b4b` | `#ffffff` + `rgba(255,255,255,0.75)` | 巨型宣言数字、认知颠覆对比卡右列 |
| ARCHITECT | `#0f172a` → `#1e293b` → `#334155` | `#ffffff` + `#cbd5e1` | 规格说明书卡、分层架构卡标题行 |
| PRAGMATIC | `#1e1b4b` | `#e2e8f0` | 代码块（terminal 风） |

**ARCHITECT 分层架构三层渐深**：
```html
<!-- 基础层 — 最深 -->
<table><tr><td style="background:#0f172a;color:#fff;">🔷 基础层：消息引擎</td></tr></table>
<!-- 中间层 — 略浅 -->
<table><tr><td style="background:#1e293b;color:#fff;">🔷 中间层：协议适配</td></tr></table>
<!-- 接口层 — 最浅 -->
<table><tr><td style="background:#334155;color:#fff;">🔷 接口层：多平台发布</td></tr></table>
```

### 13.4 Emoji 锚点技术

**原理**：在 WeChat 中，emoji 是唯一能跨平台完美渲染且不会被编辑器过滤的"图标"。大号 emoji 作为视觉锚点，效果远超 CSS icon。

**EXPLORER 探险标签云**：
```html
<table cellpadding="12" style="background:#ecfeff;text-align:center;">
  <tr>
    <td><p style="font-size:26px;margin:0;">🐦</p><p style="color:#0891b2;font-weight:700;">X/Twitter</p></td>
    <td><p style="font-size:26px;margin:0;">📱</p><p style="color:#0891b2;font-weight:700;">Reddit</p></td>
    <td><p style="font-size:26px;margin:0;">📰</p><p style="color:#0891b2;font-weight:700;">HackerNews</p></td>
  </tr>
</table>
```

**COMPANION 第一印象反应卡**：
```html
<table cellpadding="10" style="background:#ecfdf5;text-align:center;">
  <tr>
    <td><p style="font-size:28px;margin:0;">😍</p><p style="font-size:13px;color:#666;">1.6k Stars</p></td>
    <td><p style="font-size:28px;margin:0;">⭐</p><p style="font-size:13px;color:#666;">MIT 开源</p></td>
    <td><p style="font-size:28px;margin:0;">🪟</p><p style="font-size:13px;color:#666;">跨平台</p></td>
  </tr>
</table>
```

### 13.5 分割线叙事技术

**原理**：不同粗细、颜色、风格的 `<hr>` 传递不同的心理暗示。

| 风格 | hr CSS | 心理暗示 |
|------|--------|---------|
| SCHOLAR | `border-top:1px solid #d1d5db; margin:32px 0` | 论文的章节分页 |
| EXPLORER | `border-top:1px dashed #0891b2; margin:24px 0` | 地图上的探险路线 |
| PRAGMATIC | `border-top:1px solid #fecaca; margin:24px 0` | 仪表盘的告警线 |
| MENTOR | `border-top:1px dashed #d8b4fe; margin:28px 0` | 书信的折痕 |
| FUTURIST | `border-top:2px solid #1e1b4b; margin:32px 0` | 宣言的章节封面 |
| ARCHITECT | `border-top:1px solid #cbd5e1; margin:28px 0` | 蓝图的标注线 |
| COMPANION | `border-top:1px solid #d1d5db; margin:24px 0` | 聊天的"以上是历史消息" |

### 13.6 渐变色模拟（用多行 table）

WeChat 不支持 CSS 渐变。用多行 table 的不同底色模拟"伪渐变"：

**MENTOR 故事时间线（渐进紫）**：
```html
<table cellpadding="14" cellspacing="0" border="0">
  <tr><td style="background:#f5f0ff;">🚀 起点：一个普通程序员的困惑</td></tr>
  <tr><td style="background:#ede9fe;">💡 转折：他决定把所有笔记开源</td></tr>
  <tr><td style="background:#e4d4f6;">🏆 成果：350k Stars，改变了几十万人的学习路径</td></tr>
</table>
```

**ARCHITECT 规格说明书卡（伪渐变分割线不是必需的，但三层架构用了渐深背景）**：见 13.3 节。
